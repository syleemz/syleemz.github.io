Spot 인스턴스를 이용해서 비용을 낮추었습니다. 필요 시 스케일아웃을 추가로 구성하였습니다. Brexit 상황에서, AWS로 ELS 파생상품의 시나리오를 계산 적용하였습니다.

수학 모델 예측 및 Spot 인스턴스를 활용하여최소화 하였습니다. 결론적으로, 클라우드 도입 후 고객사가 기존 레거시 대비 1/10 수준으로 운영이 가능하도록 지원하였습니다.

미래에셋자산운용
웹사이트 불가용성에 대한 잠재 고객의 의구심을 극복했고, 현재는 99.999%의 가용성을 유지하는 인프라에서 운영되고 있습니다.

웹서버와 데이터 베이스 및 홈페이지 시스템의 AWS 전환 마이그레이션을 위하여 PoC를 수행하였습니다. AWS 전환을 위한 대상 홈페이지 시스템에 대한 검증을 하였습니다.

AWS 전환 시스템에 대한 안정성을 확인하는 작업을 진행했습니다. 현재 메가존클라우드는 미래에셋자산운용에 Managed Services를 제공하고 있습니다.

SM엔터테인먼트
개발자가 AWS에 직접 접속해 15분 이내에 리소스를 추가할 수 있어 개발 과정을 고속화하고, 기술팀 생산성이 극대화되었습니다

자동화된 프로세스로 AWS 아키텍처를 이용하여 중요한 파일을 관리하게 하였습니다. Amazon Elastic Compute Cloud (Amazon EC2) 인스턴스의 수를 필요 시 증감하여 트래픽에 대응하고 비용을 절감할 수 있게 되었습니다.

Amazon CloudFront 콘텐츠 전송 네트워크를 사용해 앱 사용자에게 복사본을 미리 캐시해 두고 필요할 때 신속하게 전송하고 있습니다.

직방
AWS 인프라는 국내 부동산 검색 시장에서 필요한 민첩성과 확장성을 제공했으며, 직방은 경쟁 서비스보다 세 배 이상 더 많이 다운로드되었습니다.

크기 조절이 가능한 컴퓨팅 용량을 제공하는 Amazon Elastic Compute Cloud (Amazon EC2) 인스턴스 웹/모바일 애플리케이션의 가용성을 유지하고 수요에 따라 AWS 자원을 스케일링하는 Auto Scaling

여러 클라우드 인스턴스에 트래픽을 분산하는 Elastic Load Balancing 확장 가능한 메시징 클러스터로 운영되는 Amazon Simple Queue Service (Amazon SQS) 

IGAWorks
아마존웹서비스(AWS)로 이전한 결과 아이지에이웍스 모바일 비즈니스 플랫폼은 고객 애플리케이션의 성공으로 인한 트래픽 유입에 유연하고 효율적으로 대응할 수 있는 안정적인 시스템이 되었습니다.

Amazon Simple Queue Service (Amazon SQS)를 도입해 기기 행동 데이터를 수집하고 처리하는 웹 애플리케이션과 클라우드 상의 데이터 스토리지 사이의 완충 지대를 만들어 활용했습니다.

짧은 반응시간으로 안정적인 서비스를 제공하기 위해 Key-Value 기반 데이터를 RDBMS에서 Amazon DynamoDB로 이전했습니다.
